# 🚀 Руководство по тесту максимальной производительности

## 📊 Профиль `loadshape_maxperf.py`

Stress Test для поиска максимальной производительности системы с постепенным увеличением нагрузки.

### Характеристики теста

- **Длительность:** 60 минут (3600 секунд)
- **Этапов:** 10 ступеней по 6 минут
- **Максимум пользователей:** 1200
- **Тип теста:** Stress Test / Breakpoint Test

## 🎯 Цель теста

Найти **точку деградации** системы - момент, когда:
- Response time превышает SLA
- Начинают появляться ошибки (5xx)
- RPS перестаёт расти пропорционально нагрузке
- CPU/Memory достигают предела

## 📊 Метрики для анализа

### В Locust UI:

- **RPS (Requests Per Second)** - запросов в секунду
- **Response Time** - время ответа (avg, min, max, percentiles)
- **Failures** - количество и процент ошибок
- **Users** - текущее количество виртуальных пользователей

### В Grafana:

```promql
# Response time 95th percentile
histogram_quantile(0.95, 
  rate(http_request_duration_seconds_bucket[1m])
)

# RPS по эндпоинтам
rate(http_requests_total[1m])

# CPU usage
rate(process_cpu_seconds_total[1m]) * 100

# Memory usage
process_resident_memory_bytes / 1024 / 1024
```

## 🔍 Критерии оценки

### ✅ Система работает нормально:

- Response time < 500ms (GET)
- Response time < 1000ms (POST)
- Failures < 1%
- CPU < 80%
- Memory стабильная

### ⚠️ Система начинает деградировать:

- Response time > 1000ms
- Failures 1-5%
- CPU > 80%
- Memory растёт
- RPS перестал расти

### ❌ Система перегружена:

- Response time > 5000ms
- Failures > 5%
- CPU = 100%
- Memory исчерпана
- 5xx ошибки
- RPS падает

## 📝 Анализ результатов

### После теста:

1. **Определите точку деградации:**
   ```
   Пример: На этапе 7 (600 пользователей) response time вырос с 200ms до 800ms
   ```

2. **Найдите максимальную пропускную способность:**
   ```
   Пример: Максимальный RPS = 450 req/s при 400 пользователях
   ```

3. **Выявите узкие места:**
   - CPU bottleneck?
   - Database connections?
   - Memory leak?
   - Network bandwidth?

4. **Рекомендации по оптимизации:**
   - Масштабирование (horizontal/vertical)
   - Кэширование
   - Оптимизация запросов
   - Connection pooling

## 🎯 Best Practices

1. **Мониторинг:** Всегда запускайте с включенным мониторингом (Grafana, Prometheus)
2. **Логирование:** Сохраняйте логи для последующего анализа
3. **Baseline:** Первый этап (10 пользователей) даёт baseline для сравнения
4. **Паузы:** Между тестами давайте системе восстановиться (5-10 минут)
5. **Документация:** Записывайте результаты и найденные узкие места
