# 🎨 Руководство по визуальному редактору профилей нагрузки Locust

## 📖 Описание

Визуальный редактор для создания и редактирования профилей нагрузки (LoadShape) в Locust. Позволяет визуально настраивать этапы тестирования производительности с графической визуализацией.

## 🚀 Быстрый старт

### Установка

```bash
# Установка зависимостей
pip install -r requirements-editor.txt
```

### Запуск

```bash
# Способ 1: Через скрипт (рекомендуется)
./run_editor.sh

# Способ 2: Напрямую
python3 loadshape_editor.py
```

## 🎯 Основные возможности

### 1. Визуализация профиля нагрузки

Редактор автоматически строит график нагрузки:
- **Ось X**: Время в секундах
- **Ось Y**: Количество виртуальных пользователей
- **Аннотации**: Количество пользователей на каждом этапе
- **Заполнение**: Область под графиком для лучшей визуализации

### 2. Управление этапами

#### Добавление нового этапа

1. Нажмите кнопку **➕ Добавить**
2. Заполните параметры:
   - **Длительность** - продолжительность этапа в секундах
   - **Количество пользователей** - целевое количество виртуальных пользователей
   - **Скорость запуска** - сколько пользователей запускать в секунду
   - **User Classes** - выберите один или несколько классов пользователей
3. Нажмите **✅ Сохранить**

#### Редактирование этапа

1. Выберите этап в списке
2. Нажмите кнопку **✏️ Редактировать**
3. Измените параметры
4. Нажмите **✅ Сохранить**

#### Удаление этапа

1. Выберите этап в списке
2. Нажмите кнопку **❌ Удалить**
3. Подтвердите удаление

#### Изменение порядка этапов

- **⬆️** - переместить этап выше
- **⬇️** - переместить этап ниже

### 3. Доступные User Classes

Редактор поддерживает все User Classes из вашего проекта:

| User Class | Эндпоинт | Метод | Описание |
|------------|----------|-------|----------|
| `WebUser_UC01` | `/` | GET | Главная страница |
| `WebUser_UC02` | `/contact` | GET | Страница контактов |
| `WebUser_UC03` | `/about` | GET | Страница "О нас" |
| `WebUser_UC04` | `/projects` | GET | Страница проектов |
| `WebUser_UC05` | `/contact` | POST | Отправка формы |
| `WebUser_UC05_API` | `/api/contact` | POST | JSON API |
| `WebUser_UC06` | `/metrics` | GET | Метрики Prometheus |

### 4. Статистика

Редактор автоматически рассчитывает:
- **Количество этапов** - общее число этапов в профиле
- **Общая длительность** - суммарное время теста в секундах и минутах
- **Максимум пользователей** - пиковое количество виртуальных пользователей

### 5. Предварительный просмотр

Перед сохранением можно просмотреть сгенерированный Python код:

1. Нажмите **👁️ Просмотр кода**
2. Проверьте корректность кода
3. Закройте окно предпросмотра

### 6. Сохранение

Сохранение изменений в файл `loadshape_stability.py`:

1. Нажмите **💾 Сохранить в файл**
2. Подтвердите сохранение
3. Файл будет автоматически обновлён

```
Этап 1: 0-30с, 10 пользователей (прогрев)

```

## 🔧 Технические детали

### Формат данных

Редактор работает с форматом Locust LoadShape:

```python
stages = [
    {"duration": 20, "users": 20, "spawn_rate": 2, "user_classes": [WebUser_UC01]},
    {"duration": 40, "users": 40, "spawn_rate": 2, "user_classes": [WebUser_UC03]},
    # ...
]
```

### Кумулятивная длительность

⚠️ **Важно**: В Locust параметр `duration` является **кумулятивным** (накопительным):

```python
# Правильно:
{"duration": 20, "users": 20}   # Этап длится с 0 до 20 секунд
{"duration": 40, "users": 40}   # Этап длится с 20 до 40 секунд
{"duration": 60, "users": 60}   # Этап длится с 40 до 60 секунд

# Неправильно:
{"duration": 20, "users": 20}   # Этап 1: 0-20с
{"duration": 20, "users": 40}   # ❌ Это не 20-40с, а 0-20с!
```

Редактор автоматически управляет кумулятивными значениями:
- При добавлении этапа - добавляет к предыдущему значению
- При редактировании - пересчитывает все последующие этапы
- При перемещении - корректирует порядок и значения

### Структура файла

Редактор изменяет только блок `stages` в файле `loadshape_stability.py`, сохраняя:
- Комментарии и документацию
- Импорты
- Определение класса LoadShape
- Метод tick()

## 💡 Советы по использованию

### 1. Планирование профиля

Перед созданием профиля определите:
- Цель теста (стабильность, производительность, spike test)
- Максимальное количество пользователей
- Общую длительность теста
- Какие эндпоинты тестировать

### 2. Прогрев (Warm-up)

Всегда начинайте с этапа прогрева:
- Небольшое количество пользователей (10-20)
- Короткая длительность (20-60 секунд)
- Простые GET запросы

### 3. Постепенное увеличение

Увеличивайте нагрузку постепенно:
- Не делайте резких скачков (кроме spike tests)
- Используйте разумный spawn_rate (1-5 пользователей/сек)
- Давайте системе время на стабилизацию

### 4. Тестирование POST запросов

При тестировании POST запросов:
- Используйте меньше пользователей
- Увеличьте spawn_rate
- Не забудьте `DISABLE_EMAIL=true` в `.env`

### 5. Комбинирование User Classes

Можно комбинировать несколько User Classes в одном этапе:
- Для реалистичной смешанной нагрузки
- Для проверки взаимодействия разных эндпоинтов
- Для финального этапа стабильности

## 🐛 Устранение неполадок

### Проблема: Редактор не запускается

**Решение:**
```bash
# Проверьте установку matplotlib
python3 -c "import matplotlib"

# Если ошибка - установите
pip install matplotlib
```

### Проблема: Не сохраняются изменения

**Причина:** Нет прав на запись в файл

**Решение:**
```bash
# Проверьте права
ls -la loadshape_stability.py

# Дайте права на запись
chmod u+w loadshape_stability.py
```

### Проблема: График не отображается

**Причина:** Проблемы с tkinter backend

**Решение:**
```bash
# Установите tkinter (если отсутствует)
sudo apt-get install python3-tk

# Или используйте другой backend
export MPLBACKEND=TkAgg
```

### Проблема: Некорректные кумулятивные значения

**Решение:** Редактор автоматически пересчитывает значения. Если проблема сохраняется:
1. Удалите проблемный этап
2. Добавьте его заново
3. Сохраните изменения
